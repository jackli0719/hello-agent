# 业务规格索引

> 本文件只记录已被测试覆盖的业务规格。测试注释里的 `# spec:` 应引用这里的语义；新增业务规则先补本文件，再补测试。

## ORDER

| ID        | 规格                                                                                          |
| --------- | --------------------------------------------------------------------------------------------- |
| ORDER-001 | 创建订单必须服务端校验必填项、手机号、金额、预约时间、SKU 和类目配对。                        |
| ORDER-002 | 订单金额入库单位是分，表单/业务输入金额单位是元。                                             |
| ORDER-003 | 新订单初始状态是 `pending`，订单号格式为 `O + YYYYMMDD + 当日序号`。                          |
| ORDER-004 | 合法状态流转为 `pending -> assigned -> in_service -> completed`，非终态可取消到 `cancelled`。 |
| ORDER-005 | `completed` 和 `cancelled` 是终态，不能再次流转。                                             |
| ORDER-006 | 完成或取消已分配订单必须释放师傅为 `available`。                                              |

## DISPATCH

| ID           | 规格                                                                    |
| ------------ | ----------------------------------------------------------------------- |
| DISPATCH-001 | 派单只能从 `pending` 订单开始，成功后订单变 `assigned`，师傅变 `busy`。 |
| DISPATCH-002 | 派单规则按 SKU 精确优先，其次类目兜底，同层按 `priority` 降序。         |
| DISPATCH-003 | 候选师傅必须 `status=available` 且技能覆盖规则要求。                    |
| DISPATCH-004 | 派单和状态流转必须用乐观锁防并发重复操作。                              |

## AUTH

| ID       | 规格                                                                                            |
| -------- | ----------------------------------------------------------------------------------------------- |
| AUTH-001 | 后台、师傅端和用户订单查询属于受保护路径，必须登录。                                            |
| AUTH-002 | `admin` 可访问后台管理；`worker` 只能访问 `/worker`；`customer` 只能访问 `/customer` 相关路径。 |
| AUTH-003 | 登录、登出和写操作必须校验 CSRF。                                                               |

## SERVICE

| ID          | 规格                                                        |
| ----------- | ----------------------------------------------------------- |
| SERVICE-001 | 业务编码必须是大写 ASCII 字母数字连字符，并在应用层规范化。 |
| SERVICE-002 | 新 SKU 的 `requiredSkills` 会参与派单匹配。                 |
