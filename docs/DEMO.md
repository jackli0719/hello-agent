# 三端演示脚本

> 第一版 MVP 完整演示：用户下单 → 后台派单 → 师傅履约 → 用户查询。
> 4 步走完整链路，每步预期结果都明确，演示者可逐项打勾。

---

## 准备

```bash
# 1. 初始化数据库 + 种子数据
npm run db:reset

# 2. 启动 dev server
npm run dev
# → http://localhost:3000

# 3. 打开三个浏览器标签（或窗口）：
# - Tab 1：用户端
# - Tab 2：后台管理
# - Tab 3：师傅端
```

打开首页 `/`，能看到三端入口卡片。

---

## 第一步：用户下单

**Tab 1 →** http://localhost:3000/customer

1. 顶部看到「下单 / O2O 上门服务 · MVP 演示」
2. 看到蓝色提示条「填写下方信息提交订单 → 后台分配师傅 → 师傅上门服务 → 完成后可在「查询订单」随时查看进度」
3. 在「服务品类」选 **家电清洗**（联动 SKU 下拉）
4. 在「服务 SKU」选 **空调清洗（挂机）**（自动带价格和时长）
5. 填写：
   - 姓名：`演示客户`
   - 手机号：`13912345678`（**记下这个手机号**，第四步要查）
   - 地址：`上海市浦东新区世纪大道 100 号`
   - 备注：`需要下午 2 点后上门`（可选）
6. 点「提交订单」

**预期**：

- 绿色框显示「✓ 订单已提交」
- 下面展示订单号（如 `O202606270005`）— **记下这个订单号**
- 提示「我们会尽快安排师傅与您联系」

---

## 第二步：后台派单

**Tab 2 →** http://localhost:3000/login

1. 用户名：`admin`，密码：`admin123`，点「登录」
2. 自动跳 `/dashboard`，看到 8 个统计卡
3. 点导航「订单管理」或直接访问 `/orders`

**预期**：列表最上方出现刚才的订单（`O202606270005`）：

- 客户：演示客户
- 手机号：13912345678
- 服务：空调清洗（挂机）
- 状态徽标：**待派单**（黄色）
- 「推荐」列展示系统推荐的师傅按钮（按 SKU 精确或类目兜底规则命中）

4. 点「派给他」（推荐的第一位师傅，比如李师傅 / T001）

**预期**：

- 页面刷新，订单状态变成 **已派单**（蓝色）
- 「已分配师傅」列从「未派单」变成师傅名字
- 后台 /masters 看到该师傅的 status 从 `available` 变成 `busy`

---

## 第三步：师傅履约

**Tab 3 →** http://localhost:3000/worker

1. 顶部「选择师傅」下拉框选刚才派单的师傅（如「李师傅（手机尾号 xxxx）· busy」）
2. 点「查看我的订单」

**预期**：看到刚才的订单（`O202606270005`），状态徽标 **已派单**。

3. 点卡片里的「查看详情 →」

**预期**：跳到 `/worker/orders/O202606270005?masterId=T001`：

- 顶部「订单详情 / ← 返回」
- 完整字段：订单号、客户姓名、手机、地址、服务品类、服务 SKU、金额、分配师傅、创建时间
- 蓝色大按钮「开始服务」

4. 点「开始服务」

**预期**：

- 状态徽标变 **服务中**
- 按钮变绿色「完成订单」

5. 点「完成订单」

**预期**：

- 状态徽标变 **已完成**
- 按钮消失，下面显示「✓ 订单已完成，不可再操作」灰色提示
- 后台 `/masters` 看到该师傅 status 从 `busy` 变回 `available`

---

## 第四步：用户查询状态

**回到 Tab 1 →** http://localhost:3000/customer/orders

1. 输入手机号：`13912345678`
2. 点「查询」

**预期**：看到刚才的订单（`O202606270005`），完整字段：

- 订单号
- 服务品类（家电清洗）
- 服务（空调清洗（挂机））
- 金额
- 预约时间
- 分配师傅（李师傅）
- 备注（需要下午 2 点后上门）
- 下单时间
- 状态徽标：**已完成**

## 异常演示（可选）

### 派单失败

1. 把「派单规则」里所有规则 enabled 改 false
2. 后台创建一个新订单（`/orders/new`）
3. **预期**：订单列表显示「暂无可派单师傅」红色提示

### 状态机互锁

1. 把已派单的订单点「开始服务」变服务中
2. 回到后台 `/orders` 找这单
3. **预期**：服务中订单不显示派单按钮（只能「完成订单」「取消订单」）

### 师傅端越权防护

1. 访问 `/worker/orders/O202606270005?masterId=T002`（订单归 T001 但传 T002）
2. **预期**：显示 404 页面（不告诉用户订单存在）

---

## 重置

如果演示搞乱了状态想从头来：

```bash
# 重置 DB 到 seed 默认状态（删所有订单 + 重置师傅 status）
npm run db:reset

# 然后重启 dev server（按需）
```
