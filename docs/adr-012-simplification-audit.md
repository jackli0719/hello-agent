## Context

CLAUDE.md P0-4：「业务逻辑简化即 bug」。v0.1.0 风险里说「简化业务时反问：是真不需要 / 等业务规则定义 / 我懒」——但**没系统执行**。ADR-006 是「修了一个」（完成订单不释放师傅），其他隐藏简化没扫过。

v0.2.7 节点：扫 `src/lib/` + `app/` 一遍，列出「简化即 bug」候选。

## 审计方法

1. **grep** 关键字：`TODO`、`FIXME`、`演示期`、`MVP`、`演示版`、`演示用`
2. **catch (e) 模式**：是否吞错
3. **`process.env.NODE_ENV`** 分支：dev/mock 是否合理
4. **`if (mocked)`** 模式：是不是「演示过 production」
5. **人工阅读**：基于上面 grep 结果，找最可疑的代码段

## 审计结果（v0.2.7 实跑）

### 🔴 P0 必修 — 真 bug 级

#### F1 · 客户手机号查询无验证

- **位置**：`src/lib/customer.ts:112`
- **代码**：`prisma.order.findMany({ where: { customerPhone: phone } })`
- **现状**：演示期不做手机号验证 = **任何人都能查任意手机号的订单**
- **后果**：个保法违规 + 客户隐私泄露
- **是否标注**：✅ 已标 `# MVP: 演示用，标注为「不要上线」`，但**没工具卡点**
- **v0.2.7 修复**（待做）：
  1. 加 `.env` flag：`REQUIRE_SMS_VERIFICATION=true`
  2. 当 flag = true 时，必须有 session 才返回数据
  3. 否则按现状（演示模式）
- **优先级**：**P0 必修**（任何形式上线都必须先做）

#### F2 · 师傅越权校验缺失

- **位置**：`app/worker/orders/[id]/page.tsx:35`
- **代码**：`masterId 缺省：演示期允许（不校验）`
- **现状**：调 `getOrderForWorker(id, undefined)` 不传 masterId 时 = **不加越权校验** = 拿到任意订单详情
- **后果**：师傅 A 传订单 id + 不传 masterId = 看到别人订单
- **是否标注**：✅ 已标注释，但**没工具卡点 / 404 也只覆盖找不到订单，不覆盖越权**
- **v0.2.7 修复**（待做）：
  1. masterId 缺省时一律 redirect to `/worker` 选师傅
  2. 或：当 masterId 缺省时强制要求补 session（v0.3.0 师傅登录后）
- **优先级**：**P0 必修**

### 🟡 P1 强烈建议 — 文档级 / 数据级

#### F3 · 客户预约时间兜底

- **位置**：`app/customer/actions.ts:12` + `app/customer/actions.ts:20`
- **代码**：`默认预约时间：明天 10:00（演示期简化）`
- **现状**：客户不下单时间 = 服务器默认明天 10:00
- **后果**：客户预约体验差，但功能可用
- **优先级**：**P1 建议**（演示版可接受，上线前要改）

#### F4 · 业务指标 /admin/metrics 不可信

- **位置**：`app/admin/metrics/page.tsx:118`
- **代码**：`演示版用内存计数器（dev HMR / 进程重启会清零）`
- **现状**：HMR 重启即清零 = 指标失真
- **后果**：看页面数据做决策 = 不可靠
- **是否标注**：✅ 已标
- **v0.2.7 修复**（待做）：
  1. 接 Prometheus / OTel（v0.3.0+）
  2. 或：加文件持久化（最简补救）
- **优先级**：**P1 建议**

### 🟢 P2 锦上添花 — 文档/注释级

#### F5 · `serviceArea` 字段空跑

- **位置**：`src/lib/masters.ts:93`（字段已存）+ `lib/dispatch.ts`（未读）
- **代码**：`const serviceArea = (input.serviceArea ?? "").trim();`（字段写，匹配不读）
- **现状**：派单匹配时**不读 serviceArea**（v0.2.4 ADR-008 已记录）
- **优先级**：**P2**（已知遗留 → v0.3.0+ 派单规则扩展时一起做）

#### F6 · `Master.status` 字段 UI 不暴露（已知）

- **位置**：`prisma/schema.prisma:Master.status`
- **状态**：✅ 已被处理（CLAUDE.md P0-4 案例，迁移流程无 UI 编辑入口）
- **优先级**：不需要修

## 不掩盖的发现

1. **F1 + F2 已知但未修**——v0.1.0 决策回报里就说过「上个 demo 版不验证」，v0.2.7 节点只是**首次系统写入 ADR**。
